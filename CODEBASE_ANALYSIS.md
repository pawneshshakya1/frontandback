### Key design patterns
1. **Denormalization for hot reads** — `User.pass_type`/`pass_expiry` mirrors `UserPass`; `Match.partner_tier` and `commission_rate` snapshot at event creation; `PartnerProfile` denormalizes subscription counters.
2. **Wallet stores balances as STRINGS** to avoid JS float precision bugs on currency.
3. **GeoJSON 2dsphere indexes** on `PartnerProfile.location` and `Match.location` for nearby queries.
4. **Compound unique indexes** enforce 1:1 (e.g., `Friend {user_id, friend_id}`, `UserAchievement {user_id, achievement_id}`).
5. **Tier system spans 5 models:** `PartnerProfile.partner_tier`, `TierHistory`, `ElitePass.partner_tier` (catalog), `Match.partner_tier` (snapshot), `User.mediator_application_status` (parallel concept).
6. **TTL index** for OTP auto-deletion (`{ expires_at: 1 }, expireAfterSeconds: 0`).

## 4. Backend Architecture

```

       ├── middlewares/*.middleware.js  (9 files)

```

### 4.2 Middleware stack (in order, matters)
1. `helmet()` + `noSniff`, `frameguard`, `xssFilter` — defense in depth
2. `enforceMinAppVersion` — early reject outdated clients (with 5-min in-memory cache)
3. `/api` rate limiter (5000 req/min — generous)
4. `apiResponse` — injects `res.sendSuccess` / `res.sendError`
5. `express.json` / `urlencoded` with `rawBody` capture (for Cashfree webhook signature)
6. `cors` (default `*` — see security §7)
7. `/uploads` static
8. `sanitizeInput` (strips `$`-prefix keys, `<script>`, `javascript:`)
9. `auditMiddleware` (logs ≥400 to `SecurityAudit`)

### 4.3 Rate-limit policies
- `authLimiter`: 20 / 15min
- `strictLimiter`: 100 / 15min
- `walletLimiter`: 30 / 5min
- `friendLimiter`: 50 / 1hr
- `webhookLimiter`: 100 / 1min

### 4.4 Bootstrap & lifecycle (server.js)
- Hard-fails on missing `MONGO_URI` or `JWT_SECRET`
- Connects to Mongo **before** `app.listen()`
- `SIGTERM`/`SIGINT` handlers with 10s forced shutdown
- `unhandledRejection` only logs (does not exit) — debatable choice
- `uncaughtException` exits (correct)

## 6. Cross-Cutting Architectural Observations

1. **Two parallel SSE implementations** — `services/sse.ts` (singleton class) and `hooks/useSSE.ts` (stateful hook). Pick one.
2. **Two parallel notification sources** — Firestore (push) and REST (`notificationAPI`). Document the split.
3. **Polling + SSE race** — `useMatchPolling` and SSE both update match state; can flicker.
4. **No state persistence** anywhere — hot reload / app restart loses the session.
5. **No request cancellation** / no React-Query-style cache.
6. **Loose typing** — `any` in many hot paths despite `tsconfig.strict: true`.
7. **Empty test directories** — `__tests__/` under `services/` and `hooks/` planned but unwritten.
8. **Auth/Wallet/Match stores are decoupled** — joining a match changes wallet balance, but the wiring is left to components.
9. **Three-layer backend is clean** — routes are thin, controllers call services, services own business logic.
10. **Multi-tenancy by role** consistently applied at the route level.
11. **Webhook signature verification** uses `crypto.timingSafeEqual` + `rawBody` capture (best practice).
12. **Idempotency** for webhooks via `CashfreeWebhookLog` dedup (best practice).

## 11. What Works Well (don't break)

- ✅ **Clean layered architecture** (routes → controllers → services → models).
- ✅ **Webhook security** — `crypto.timingSafeEqual` + `rawBody` + idempotency log.
- ✅ **Defensive startup** — server hard-fails on missing critical env vars.
- ✅ **TTL index** for OTP auto-deletion.
- ✅ **Geo 2dsphere indexes** for nearby partner/event queries.
- ✅ **String balances in Wallet** — avoids float precision issues.
- ✅ **Universal 16px horizontal padding** — consistently honored across all reviewed screens.
- ✅ **PopupModal / usePopup** — clean reusable component.
- ✅ **Denormalized hot data** (UserStats, UserSpendAnalytics, PartnerProfile counters) — good for read perf.
- ✅ **mongodb-memory-server** test setup is clean and hermetic.
- ✅ **No circular dependencies** in either codebase.
